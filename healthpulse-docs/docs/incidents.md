# Incident Log

_Document every issue, outage, or unexpected problem you encountered. This is how real SRE teams learn from failures._

---

## Incident Template

_Copy this block for each incident._

### INC-001: _Short title_

| Field | Value |
|-------|-------|
| **Date** | _YYYY-MM-DD_ |
| **Severity** | _Low / Medium / High / Critical_ |
| **Environment** | _Dev / UAT / QA / Prod_ |
| **Duration** | _e.g., 25 minutes_ |
| **Detected By** | _Datadog alert / Manual / User report_ |
| **Resolved By** | _Team member name_ |

**What happened:**

_Describe the incident in 2-3 sentences._

**Root cause:**

_What actually caused the problem._

**Resolution:**

_What you did to fix it, including commands run._

**Lessons learned:**

_What will you do differently to prevent this in the future?_

---

!!! info "Why Track Incidents?"
    In a real DevOps job, you'll write postmortems after every incident. Practicing this skill now — documenting what broke, why, and what you changed — is more valuable than the fix itself. Interviewers love candidates who can walk through a real incident response.
